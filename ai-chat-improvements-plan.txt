# AI Chatbox Improvements Plan

## 1. Advanced Search Improvements
- **Conversational Filter State**: Pass existing filters back to the prompt (like Add Flight does) to enable relative refinements (e.g., "Change the terminal to 5" or "Actually, search tomorrow instead"). Currently, it treats every search iteratively.
- **Results Summarization**: Instead of just applying filters silently, the AI could read the results (or the backend could inject result stats) and reply verbally, e.g., "I found 4 delayed BA flights for you."
- **Filter Negation**: Make sure the AI understands requests to clear filters, such as "Remove the terminal filter" or "Show all statuses again."

## 3. Dashboard Integration (Home/Index)
**Goal:** Make the Dashboard an interactive command center.
- **Real-Time DOM Highlighting**: Instead of filtering via the server, querying the AI to "Highlight delayed arrivals" returns a directive to the JS layer, which adds CSS highlight classes to matching table rows in real-time.
- **Quick Statistics**: Supply the prompt with a cached JSON snapshot of the visible dashboard so the AI can answer data-driven questions. "What's our busiest terminal right now?" or "How many total departures are left today?"
- **Direct Navigation**: "Find me flight BA123" could tell the client to scroll to that row and pulse/highlight it.

## 4. Flight Details Integration (Details/Modal)
**Goal:** Provide deep context and actions for a specific flight.
- **Context Injection**: Seed the DeepSeek prompt with the selected flight's JSON details (status, predicted times, destination) upon opening the modal.
- **Generate Announcements**: "Draft a PA announcement for this delay". The AI reads the destination and revised time, drafting a professional public address message.
- **External "Tool" Simulation**: Allow the user to ask "What's the weather at the destination?" by having the backend identify the destination IATA, fetch weather data, and pass it immediately to the chat.
